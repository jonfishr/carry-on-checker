Software Architecture & Implementation Guide

🎯 Architecture Overview

This is a static, serverless architecture designed for minimal recurring costs while maintaining excellent performance and user experience. The entire application runs in the browser with no backend servers required.

🌐 Frontend Layer (Browser)

HTML + CSS + Vanilla JavaScript

Mobile-first responsive design with instant client-side calculations. No framework overhead means faster load times and better performance on mobile devices.

💾 Data Layer (Static JSON)

JSON files embedded or loaded at startup

Two JSON files: airlines.json (airline overhead bin dimensions) and luggage.json (luggage library). Total size: ~5-10KB compressed.

☁️ Hosting Layer

Cloudflare Pages, Netlify, or GitHub Pages

Static hosting with global CDN, automatic SSL, and zero-config deployments. All offer generous free tiers perfect for this use case.

✨ Core Features

  • Bag Size Input: Users enter height, width, depth (with cm/inches toggle)
  • Instant Compatibility Check: Real-time calculation showing which airlines accept the bag
  • Airline Score: "Fits X out of Y airlines (Z%)" display
  • Luggage Library: Browse pre-vetted bags with airline compatibility
  • Responsive Design: Mobile-first UI that works great on all devices
  • Search & Filter: Find bags by brand, size, or airline compatibility
  • Share Results: Generate shareable links for specific bag configurations

🚀 User Flow

Step 1: Landing Page

User sees two primary options: "Check My Bag" or "Browse Luggage Library"

Step 2: Input Dimensions

User enters bag dimensions or selects from library. Unit selector (cm/inches) prominently displayed.

Step 3: View Results

Instant display of compatible airlines with visual indicators (✅/❌). Airlines sorted by compatibility score.

User can adjust dimensions, view airline notes, or browse alternative bags from the library.

🔄 Data Update Strategy

Easy Updates Without Redeployment

Since airline rules change periodically, consider one of these strategies:

  • Option 1 (Simplest): Edit JSON files in your GitHub repo and commit. Cloudflare Pages auto-deploys in ~30 seconds.
  • Option 2 (Recommended): Use Airtable as a free CMS. Fetch JSON from Airtable's API on page load (adds ~100ms latency but allows non-technical updates).
  • Option 3 (Advanced): Build a simple admin interface using Cloudflare Workers + KV storage (still free tier).

🎓 Why This Architecture?

Cost: Hosting is completely free. Your only cost is a domain name (~$1/month) if you want one.

Performance: Static sites served from a CDN are the fastest possible option. Your site will load in under a second worldwide.

Scalability: CDNs handle millions of visitors effortlessly. You'll never worry about server costs scaling with traffic.

Simplicity: No backend to maintain, no databases to manage, no server updates. Just HTML, CSS, and JavaScript.

Reliability: Static sites have 99.99%+ uptime. No databases to crash or servers to go down.

Mobile-First: Lightweight and fast = perfect for mobile users checking bags before their flight.
